## 1. 这个项目是干什么的？

简单来说：**npm-downloader 是一个 npm 依赖包离线下载工具。**

### 什么场景下会用到？

- 你的开发电脑**有网络**，但部署的服务器**没有网络**（内网环境）
- 你想把一个项目依赖的所有 npm 包**提前下载好**，拷贝到离线环境中使用
- 你想批量下载某个 npm 包及其所有依赖项

### 它能做什么？

| 功能 | 说明 |
|------|------|
| 上传 lockfile | 上传 `pnpm-lock.yaml` 文件，自动解析并下载里面列出的所有依赖 |
| 指定包名下载 | 输入一个 npm 包名（如 `lodash@4.17.21`），自动下载它及所有依赖 |
| 实时日志 | 下载过程中可以看到实时的日志输出，知道哪些包下载成功了，哪些失败了 |
| 打包下载 | 所有包下载完成后，自动打包成 ZIP 文件，你可以一键下载 |
| 历史记录 | 之前的下载任务会被记录下来，你可以随时查看和重新下载 |

## 2. 开始之前：你需要准备什么

### 必需软件

| 软件 | 最低版本 | 如何检查是否已安装 | 如何安装 |
|------|---------|-------------------|---------|
| **Node.js** | >= 18 | 打开终端输入 `node --version` | 访问 https://nodejs.org/ 下载安装 |
| **pnpm** | 任意版本 | 打开终端输入 `pnpm --version` | 安装好 Node.js 后，终端输入 `npm install -g pnpm` |

### 操作系统

- Windows 10/11（本项目提供了 Windows 专用脚本）
- 也支持 Linux/macOS（有 shell 脚本）

### 检查环境

打开终端（Windows 上可以用 PowerShell 或 CMD），依次输入：

```bash
node --version
# 应该输出类似 v20.x.x 或 v22.x.x，只要 >= 18 就行

pnpm --version
# 应该输出类似 10.x.x
```

如果 `pnpm` 没有安装，运行：

```bash
npm install -g pnpm
```

## 3. 安装项目

### 第一步：打开终端，进入项目目录

```bash
cd C:\你的路径\npm-downloader
```

### 第二步：安装依赖

```bash
pnpm install
```

> **这会做什么？** 项目用到了很多第三方库（比如 Express 用来做 Web 服务器，Vue 用来做前端界面），这一步会把它们全部下载到本地的 `node_modules` 文件夹中。
>
> **需要等多久？** 通常 1-3 分钟，取决于你的网络速度。
>
> **如果报错？** 可能是网络问题，可以尝试配置 npm 镜像源：
> ```bash
> pnpm config set registry https://registry.npmmirror.com
> pnpm install
> ```

## 4. 启动项目

项目分为**前端**（Web 界面）和**后端**（API 服务器）两部分，需要同时启动。

### 方式一：开发模式（推荐新手使用）

开发模式的好处是：你修改了代码，服务器会**自动重启**，不用手动重新启动。

#### Windows 用户

**最简单的方式：双击 `start-dev.bat`**

或者手动在终端中操作：

```bash
# 步骤 1：启动后端（打开一个新的终端窗口）
pnpm -C packages/server dev
# 后端会在 http://localhost:3002 运行
# 不要关闭这个终端窗口！

# 步骤 2：再打开一个新的终端窗口，启动前端
pnpm -C packages/client dev
# 前端会在 http://localhost:3000 运行
```

#### 也可以用 Turborepo 一键启动

```bash
turbo run dev
# 这会同时启动前端和后端
```

启动成功后：
- 前端地址：http://localhost:3000
- 后端地址：http://localhost:3002

打开浏览器，访问 **http://localhost:3000** 就能看到界面了。

### 方式二：生产模式

生产模式需要先**构建**（把 TypeScript 代码编译成 JavaScript），然后再启动。

```bash
# 第一步：构建所有包
pnpm run build

# 第二步：启动后端
cd packages/server
pnpm start
# 后端运行在 http://localhost:3002

# 第三步（新终端窗口）：启动前端
cd packages/client
pnpm run serve:static
# 前端运行在 http://localhost:3001
```

> **注意**：生产模式下前端端口是 **3001**，开发模式下是 **3000**，这是正常的。

### 方式三：PowerShell 脚本（Windows 推荐）

项目提供了一个功能完善的 PowerShell 管理脚本 `npm-downloader.ps1`。

```powershell
# 启动生产模式（会自动安装依赖 + 构建 + 启动）
.\npm-downloader.ps1 start

# 启动开发模式
.\npm-downloader.ps1 dev

# 停止服务
.\npm-downloader.ps1 stop

# 重启服务
.\npm-downloader.ps1 restart

# 查看运行状态
.\npm-downloader.ps1 status

# 查看日志
.\npm-downloader.ps1 logs          # 列出所有日志文件
.\npm-downloader.ps1 logs server   # 查看后端日志（最近 50 行）
.\npm-downloader.ps1 logs client   # 查看前端日志（最近 50 行）

# 仅构建（不启动）
.\npm-downloader.ps1 build

# 仅安装依赖
.\npm-downloader.ps1 install
```

> **如果 PowerShell 提示执行策略错误：**
> ```powershell
> Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
> ```

### 方式四：Docker（进阶）

如果你安装了 Docker，可以用 Docker 一键运行：

```bash
# 构建并启动
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止
docker-compose down
```

Docker 启动后：
- 前端：http://localhost:3001
- 后端：http://localhost:3002

> 下载的文件会保存在 Docker 容器外的 `./downloads` 目录中。

## 5. 使用 Web 界面下载 npm 包

打开浏览器，访问前端地址（开发模式：http://localhost:3000，生产模式：http://localhost:3001）。

### 页面布局

页面分为左右两部分：
- **左侧**（320px 宽）：操作面板，包含上传区域和包名输入区域
- **右侧**：下载历史记录列表

### 方式 A：上传 lockfile 文件

这是最常用的方式，适用于"我要把一个已有项目的所有依赖都下载下来"的场景。

**操作步骤：**

1. 在左侧面板找到 **"Lockfile Upload"** 区域
2. 将你的 `pnpm-lock.yaml` 文件**拖拽到虚线框**内，或者**点击虚线框**选择文件
   - 文件必须是 `pnpm-lock.yaml` 格式（如果你用的是 npm 或 yarn，暂时不支持）
3. 点击 **"Upload"** 按钮
4. 系统开始解析并下载依赖：
   - 你会看到上传进度
   - 下载开始后，左侧会出现 **"Current Task"** 卡片，显示实时进度
   - 点击 "View Logs" 按钮可以打开日志查看器，看到每个包的下载情况
5. 下载完成后，历史记录卡片会出现 **"Download ZIP"** 按钮
6. 点击 **"Download ZIP"** 即可下载包含所有 npm 包的 ZIP 压缩包

**什么是 `pnpm-lock.yaml`？**
这是 pnpm 包管理器自动生成的文件，记录了项目依赖的每一个包的精确版本和下载地址。它通常位于你项目的根目录下。

### 方式 B：直接输入包名下载

适用于"我只需要下载某一个包（及其依赖）"的场景。

**操作步骤：**

1. 在左侧面板找到 **"Single Package"** 区域
2. 在输入框中输入包名，支持以下格式：
   - `lodash`（下载最新版）
   - `lodash@4.17.21`（下载指定版本）
   - `@vue/core`（带 scope 的包名也可以）
3. 点击 **"Download"** 按钮
4. 系统会自动解析该包的所有依赖，并并发下载（最多同时下载 10 个）
5. 下载完成后，和历史记录中一样可以下载 ZIP 文件

> **提示**：系统会自动递归下载目标包的所有依赖、依赖的依赖……确保你拿到的是完整的依赖树。

## 6. 使用命令行工具（CLI）

如果你不想用 Web 界面，也可以直接用命令行下载。这是 `@npm-downloader/core` 包提供的功能。

```bash
# 先构建 core 包
cd packages/core
pnpm run build

# 使用 CLI 下载
node dist/cli.js --lockfile /path/to/pnpm-lock.yaml

# 只下载前 10 个包（测试用）
node dist/cli.js --lockfile /path/to/pnpm-lock.yaml --limit 10

# 查看帮助
node dist/cli.js --help
```

**CLI 参数说明：**

| 参数 | 缩写 | 说明 | 示例 |
|------|------|------|------|
| `--lockfile` | `-l` | 指定 `pnpm-lock.yaml` 文件路径，或其所在目录 | `--lockfile ./my-project/` |
| `--limit` | `-n` | 只下载前 N 个包 | `--limit 5` |
| `--help` | `-h` | 显示帮助信息 | |

下载的 `.tgz` 文件会保存在当前目录的 `downloadTgz/` 文件夹中。


### 关键概念解释

| 术语 | 解释 |
|------|------|
| **Monorepo** | 把多个相关的包放在同一个代码仓库里管理。这个项目有 4 个包：core、server、client、types |
| **pnpm workspaces** | pnpm 提供的 monorepo 管理方案，让包之间可以互相引用 |
| **Turborepo** | 一个构建工具，可以智能地安排多个包的构建顺序，还支持缓存加速 |
| **SSE** | Server-Sent Events，服务器主动向浏览器推送数据的技术。本项目用它在下载过程中实时推送日志 |
| **tgz** | npm 包的压缩格式。每个发布到 npm 的包本质上就是一个 `.tgz` 压缩文件 |
| **lockfile** | 锁定文件，记录了项目所有依赖的精确版本号，确保每次安装的结果一致 |

## 11. 常见问题与排错

### Q1: `pnpm install` 报错，提示网络超时

**原因**：默认的 npm 源在国外，国内访问可能很慢。

**解决方案**：切换到国内镜像源：

```bash
pnpm config set registry https://registry.npmmirror.com
pnpm install
```

### Q2: 启动时提示端口被占用

```
Error: listen EADDRINUSE: address already in use :::3002
```

**原因**：之前启动的服务没有正确关闭，端口还在被占用。

**解决方案**：

```bash
# 方法一：使用 stop.bat
.\stop.bat

# 方法二：手动查找并关闭占用端口的进程
# 查看哪个进程占用了 3002 端口
netstat -ano | findstr ":3002"
# 找到 PID 后，强制结束进程
taskkill /PID <PID号> /F
```

### Q3: 构建失败

**解决方案**：

```bash
# 尝试清理后重新构建
pnpm run clean
pnpm install
pnpm run build
```

### Q4: 前端页面打开是空白

**可能原因**：
1. 后端服务没有启动 → 先启动后端
2. 端口号弄错了 → 开发模式前端是 3000，生产模式是 3001
3. 浏览器缓存 → 按 `Ctrl + Shift + R` 强制刷新

### Q5: 上传 lockfile 后没有反应

**可能原因**：
1. 后端服务没有启动或已崩溃 → 检查后端终端有没有报错
2. 文件格式不对 → 确保上传的是 `pnpm-lock.yaml`，不是 `package-lock.json` 或 `yarn.lock`
3. lockfile 中没有依赖 → 确保你的项目确实有安装依赖

### Q6: 下载的 ZIP 文件找不到

下载完成后：
- 通过 Web 界面点击历史记录卡片中的 **"Download ZIP"** 按钮下载
- ZIP 文件在后端的临时目录中，**1 小时后会自动删除**（清理机制）
- 如果找不到，可能已被自动清理，需要重新下载

### Q7: Docker 启动失败

```bash
# 查看 Docker 日志
docker-compose logs -f

# 重新构建（不使用缓存）
docker-compose build --no-cache
docker-compose up -d
```

### Q8: PowerShell 脚本执行策略错误

```
.\npm-downloader.ps1 : 无法加载文件，因为在此系统上禁止运行脚本
```

**解决方案**：

```powershell
# 以管理员身份运行 PowerShell，执行：
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

## 13. API 接口一览

如果你是开发者，想了解后端提供了哪些 API 接口：

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/api/upload` | 上传 lockfile 并开始下载 |
| `POST` | `/api/download-package` | 指定包名开始下载 |
| `GET` | `/api/history` | 获取所有历史记录 |
| `DELETE` | `/api/history/:taskId` | 删除某条历史记录 |
| `GET` | `/api/logs/:taskId` | 获取某个任务的日志 |
| `GET` | `/api/logs/:taskId/stream` | SSE 实时日志流（浏览器监听） |
| `GET` | `/api/download/:taskId` | 下载 ZIP 文件 |
| `GET` | `/api/task/:taskId` | 查询任务状态和进度 |
| `GET` | `/health` | 健康检查（返回 `{ "status": "ok" }`） |

### API 使用示例

```bash
# 上传 lockfile 并下载
curl -X POST http://localhost:3002/api/upload \
  -F "file=@/path/to/pnpm-lock.yaml"

# 指定包名下载
curl -X POST http://localhost:3002/api/download-package \
  -H "Content-Type: application/json" \
  -d '{"packageName": "lodash@4.17.21"}'

# 查看历史记录
curl http://localhost:3002/api/history

# 查看任务状态
curl http://localhost:3002/api/task/1710000000000

# 下载 ZIP
curl -O http://localhost:3002/api/download/1710000000000

# 健康检查
curl http://localhost:3002/health
```

## 快速启动速查表

| 场景 | 命令 |
|------|------|
| 首次安装 | `pnpm install` |
| 开发模式启动 | 双击 `start-dev.bat`，或 `turbo run dev` |
| 生产模式启动 | 双击 `start.bat`，或先 `pnpm run build` 再手动启动 |
| PowerShell 启动 | `.\npm-downloader.ps1 dev` |
| 停止服务 | 双击 `stop.bat`，或 `.\npm-downloader.ps1 stop` |
| 查看状态 | 双击 `status.bat`，或 `.\npm-downloader.ps1 status` |
| 查看日志 | `.\npm-downloader.ps1 logs server` |
| 构建项目 | `pnpm run build` |
| 清理构建产物 | `pnpm run clean` |
| 运行测试 | `pnpm run test` |
| Docker 启动 | `docker-compose up -d` |
