# 轻量级路线图 (Roadmap)

本文档定义了 `npm-downloader` 作为**个人/本地工具**的优化方向，核心目标是：**易用、稳定、零配置**。

## 第一阶段：体验与稳定性 (近期)
重点是修复已知问题，确保工具在本地运行时的可靠性。

### 1. 本地数据持久化 (零配置)
- **目标**：重启服务后，历史记录依然存在。
- **方案**：使用简单的 **JSON 文件** 存储 (例如 `lowdb` 或直接读取/写入 `.json` 文件)。不需要安装任何数据库。
- **功能**：
  - 记录下载过的包名、时间和任务状态。
  - 支持清理历史记录。

### 2. 代码重构与类型共享
- **目标**：消除重复代码，提高维护效率。
- **任务**：
  - 修复客户端 `app.vue` 中硬编码的 `localhost:3002`，使其真正支持环境变量。
  - 提取共享类型定义，避免前后端接口不一致导致运行错误。

### 3. 下载成功率优化
- **功能**：
  - **自动重试**：网络抖动导致单个包下载失败时，自动进行 2-3 次重试。
  - **跳过机制**：如果任务中部分包下载失败，允许生成包含“部分成功”包的 ZIP，并附带失败清单。

## 第二阶段：功能完善 (中期)

### 1. 全局本地缓存 (省流量/提速)
- **功能**：在本地设置一个缓存目录。如果之前下载过 `react@18.2.0`，下次下载直接从缓存读取，不再请求网络。
- **价值**：极大提升重复下载的速度，且支持弱网/离线操作。

### 2. 企业私有仓库支持
- **功能**：支持读取本地 `.npmrc` 或通过环境变量配置 Registry 地址。
- **价值**：能够下载公司内部的私有依赖包。

### 3. 部署优化 (一键运行)
- **任务**：
  - 提供 `docker-compose` 脚本，实现“下载即运行”。
  - 优化镜像体积，确保轻量化。

## 第三阶段：长远规划 (可选)

### 1. 更多格式支持
- 支持 `package-lock.json` (npm) 和 `yarn.lock` (yarn)。

### 2. 简易配置界面
- 在 UI 上可以直接修改并发数、缓存目录等设置。
