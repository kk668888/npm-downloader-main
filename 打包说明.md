## 1. 打包原理

桌面应用的打包分为 3 个阶段：

```
┌─────────────────────────────────────────────────────────┐
│  阶段 1：构建各个子包                                      │
│  ├── types  → tsc 编译 → packages/types/dist/            │
│  ├── core   → tsc 编译 → packages/core/dist/             │
│  ├── server → tsc 编译 → packages/server/dist/           │
│  └── client → vite 构建 → packages/client/dist/          │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  阶段 2：esbuild 打包服务器                                │
│  server/dist/app.js → esbuild bundle → build/server/app.js│
│  （将 Express 及所有 npm 依赖打包成单个 JS 文件）            │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  阶段 3：electron-builder 生成安装包                       │
│  ├── Electron 运行时                                      │
│  ├── src/main.js（主进程）                                 │
│  ├── extraResources/server/app.js（打包后的服务器）          │
│  ├── extraResources/client/dist/（前端静态文件）             │
│  └── 生成 NSIS 安装程序（.exe）                             │
└─────────────────────────────────────────────────────────┘
```

**关键设计：**

| 设计点 | 原因 |
|--------|------|
| esbuild 打包服务器为单文件 | 避免 pnpm monorepo 的 `node_modules` 符号链接在 electron-builder 中不可用 |
| 服务器作为子进程 fork 运行 | Express 需要 Node.js 运行时，不能直接在 Electron renderer 中运行 |
| DESKTOP_MODE 环境变量 | 告诉 Express 同时服务 API + 前端静态文件（不需要单独的前端服务器） |
| extraResources 而非 asar | 服务器文件通过 `process.resourcesPath` 访问，不放入 asar 包 |

## 2. 环境准备

### 必需软件

| 软件 | 最低版本 | 检查命令 |
|------|---------|---------|
| **Node.js** | >= 18 | `node --version` |
| **pnpm** | >= 8 | `pnpm --version` |
| **Windows** | 10/11 | — |

> **磁盘空间**：Electron 下载 + 构建产物约需 500MB 空间。

### 安装项目依赖

```bash
# 在项目根目录执行
pnpm install
```

> 如果安装 Electron 下载缓慢，可以配置镜像：
> ```bash
> # 设置 Electron 下载镜像（淘宝源）
> export ELECTRON_MIRROR="https://npmmirror.com/mirrors/electron/"
> # Windows PowerShell 用这个：
> $env:ELECTRON_MIRROR = "https://npmmirror.com/mirrors/electron/"
> ```

## 3. 快速打包

**一条命令搞定：**

```bash
# 在项目根目录执行
pnpm build:desktop
```

这条命令会自动完成：构建所有子包 → esbuild 打包服务器 → electron-builder 生成安装包。

完成后在 `packages/desktop/release/` 目录下找到安装包。

## 4. 分步打包（详细流程）

如果你想理解每一步在做什么，或者排查某个步骤的错误，可以按以下步骤手动执行。

### 步骤 1：构建 types 包（共享类型）

```bash
cd packages/types
npx tsc
```

**产物**：`packages/types/dist/index.js`

**为什么先构建**：server 和 client 都依赖 types 包的类型定义。

### 步骤 2：构建 core 包（核心逻辑）

```bash
cd packages/core
npx tsc
```

**产物**：`packages/core/dist/`

### 步骤 3：构建 server 包（后端服务）

```bash
cd packages/server
npx tsc
```

**产物**：`packages/server/dist/app.js` 及其他编译后的 JS 文件

### 步骤 4：构建 client 包（前端界面）

```bash
cd packages/client
pnpm run build
```

**产物**：`packages/client/dist/`（包含 `index.html`、JS、CSS 等静态文件）

> 这一步会先运行 `vue-tsc` 类型检查，再执行 `vite build`。如果类型检查报错但不影响功能，可以临时改为 `vite build` 跳过检查。

### 步骤 5：esbuild 打包服务器为单文件

```bash
cd packages/desktop
pnpm run bundle:server
```

**产物**：`packages/desktop/build/server/app.js`（单个文件，约 2-5 MB）

**这一步做什么**：将 `packages/server/dist/app.js` 及其所有 `node_modules` 依赖（Express、pacote、archiver 等）打包成一个独立的 JS 文件。这样 electron-builder 不需要处理复杂的 pnpm 符号链接。

### 步骤 6：electron-builder 生成安装包

```bash
cd packages/desktop
npx electron-builder --win
```

**产物**：`packages/desktop/release/` 目录下的安装包

> 如果只想生成免安装版（调试用），用 `npx electron-builder --win --dir`。

## 5. 打包产物说明

### 目录结构

```
packages/desktop/
├── build/
│   └── server/
│       └── app.js                  ← esbuild 打包后的服务器（单个文件）
│
└── release/
    ├── npm-downloader Setup 1.0.0.exe  ← NSIS 安装包（分发给用户）
    ├── win-unpacked/                    ← 免安装版（直接运行）
    │   ├── npm-downloader.exe          ← 主程序
    │   ├── resources/
    │   │   ├── app.asar                ← Electron 主进程代码
    │   │   ├── server/
    │   │   │   └── app.js             ← 打包后的服务器
    │   │   ├── client/
    │   │   │   └── dist/              ← 前端静态文件
    │   │   └── assets/                ← 图标等资源
    │   └── *.dll                       ← Electron/Chromium 运行时
    └── builder-effective-config.yaml   ← 实际使用的构建配置
```

### 安装包信息

| 项目 | 值 |
|------|-----|
| 安装包格式 | NSIS（支持自定义安装路径） |
| 目标架构 | x64（64 位） |
| 安装包大小 | 约 80-120 MB（含 Electron 运行时） |
| 应用 ID | `com.npm-downloader.desktop` |

### 运行时数据存储

桌面应用的所有用户数据存储在 Electron 的 `userData` 目录：

```
%APPDATA%/npm-downloader/
├── data/
│   ├── history.json          ← 下载历史记录
│   └── logs/                 ← 任务日志文件
├── uploads/                  ← 上传的 lockfile 文件
└── temp/                     ← 临时文件（ZIP 包等，1 小时自动清理）
```

## 6. 自定义图标

### 图标要求

| 格式 | 用途 | 尺寸 |
|------|------|------|
| `icon.ico` | Windows | 256x256（建议包含 16/32/48/256 多尺寸） |
| `icon.png` | 通用 | 512x512 |
| `icon.icns` | macOS | 512x512 |

### 放置位置

将图标文件放到 `packages/desktop/assets/` 目录下：

```
packages/desktop/assets/
├── icon.ico    ← Windows 图标
├── icon.png    ← 通用 PNG 图标
└── icon.icns   ← macOS 图标（可选）
```

> **如果没有图标**：electron-builder 会使用默认的 Electron 图标。打包不会失败，只是图标不好看。

### 制作图标

推荐使用在线工具生成多格式图标：

1. 准备一张 512x512 的 PNG 图片
2. 使用工具（如 [electron-icon-builder](https://github.com/nicedoc/electron-icon-builder)）生成 `.ico`、`.icns` 等格式
3. 将生成的文件放入 `assets/` 目录

## 7. 常见问题与排错

### Q1: `pnpm build:desktop` 报错 "Entry point not found"

**错误信息**：
```
[bundle-server] Entry point not found: packages/server/dist/app.js
[bundle-server] Please run "pnpm -C packages/server build" first
```

**原因**：服务器还没有构建，`dist/` 目录不存在。

**解决方案**：
```bash
# 先构建所有子包
pnpm run build
# 再打包桌面应用
pnpm build:desktop
```

### Q2: Electron 下载超时

**错误信息**：
```
Error: Request timed out after 30000ms
```

**原因**：Electron 二进制文件较大（约 80MB），国内下载可能很慢。

**解决方案**：
```bash
# 配置 Electron 镜像源
# Linux/Mac：
export ELECTRON_MIRROR="https://npmmirror.com/mirrors/electron/"

# Windows PowerShell：
$env:ELECTRON_MIRROR = "https://npmmirror.com/mirrors/electron/"

# 然后重新安装
pnpm install
```

### Q3: vue-tsc 类型检查报错导致 client 构建失败

**原因**：Vue 组件的类型定义可能不完整。

**解决方案**：
```bash
# 临时跳过类型检查，只做 vite build
cd packages/client
npx vite build

# 然后继续打包桌面应用
cd ../desktop
pnpm run bundle:server
npx electron-builder --win
```

### Q4: electron-builder 报错 "icon.ico does not exist"

**原因**：没有提供 Windows 图标文件。

**解决方案**：

方案 A：添加图标文件到 `packages/desktop/assets/icon.ico`

方案 B：暂时移除图标配置（编辑 `electron-builder.yml`，删除 `win.icon` 行）

### Q5: 打包后的应用启动后白屏

**可能原因**：

1. **前端构建产物不完整** — 检查 `packages/client/dist/` 目录下是否有 `index.html`
2. **服务器 bundle 失败** — 检查 `packages/desktop/build/server/app.js` 是否存在且大小合理（> 1MB）
3. **路径问题** — 打开 DevTools（按 F12）查看控制台错误

**调试方法**：
```bash
# 先构建免安装版（可以看控制台输出）
cd packages/desktop
pnpm build:desktop:dir

# 运行免安装版
./release/win-unpacked/npm-downloader.exe
```

### Q6: 安装后运行报错 "Server crashed with exit code 1"

**原因**：服务器 bundle 可能缺少某些依赖或存在路径问题。

**调试方法**：

1. 打开应用后按 `F12` 打开 DevTools
2. 在 Console 中查看 `[Server]` 开头的日志
3. 常见问题：
   - 端口 3002 被占用 → 关闭其他占用端口的程序
   - 数据目录权限不足 → 以管理员身份运行

### Q7: 构建速度慢

**优化建议**：

```bash
# 1. 使用 Turborepo 缓存（如果之前构建过，会自动跳过未修改的包）
pnpm run build

# 2. 只构建免安装版（跳过 NSIS 打包，速度快很多）
pnpm build:desktop:dir

# 3. 如果只改了前端代码，只需重新构建 client + 打包
cd packages/client && pnpm run build
cd ../desktop && npx electron-builder --win --dir
```

## 8. 高级配置

### 修改应用名称

编辑 `packages/desktop/electron-builder.yml`：

```yaml
productName: "你的应用名称"  # 显示在安装包和窗口标题
```

同时修改 `packages/desktop/src/main.js` 中的 `title`：

```javascript
title: '你的应用名称',
```

### 修改应用 ID

编辑 `packages/desktop/electron-builder.yml`：

```yaml
appId: com.your-company.your-app
```

### 修改端口号

桌面应用默认使用 3002 端口。如果需要修改：

编辑 `packages/desktop/src/main.js`：

```javascript
const port = 3002  // 改为你想要的端口
```

### 支持 macOS / Linux

当前配置已预留了 macOS 和 Linux 的构建配置，但需要额外步骤：

```bash
# macOS（需要在 macOS 上运行）
cd packages/desktop
pnpm build:mac

# Linux（需要在 Linux 上运行）
cd packages/desktop
pnpm build:linux
```

> **注意**：跨平台打包有限制。建议在目标平台上构建。

### 修改安装程序行为

编辑 `packages/desktop/electron-builder.yml` 中的 `nsis` 部分：

```yaml
nsis:
  oneClick: true                    # 改为 true 则一键安装（不显示安装路径选择）
  allowToChangeInstallationDirectory: false
  createDesktopShortcut: true       # 是否创建桌面快捷方式
  createStartMenuShortcut: true     # 是否创建开始菜单快捷方式
```

### 构建配置文件参考

```
packages/desktop/
├── package.json              ← 构建脚本定义
├── electron-builder.yml      ← electron-builder 配置
├── scripts/
│   └── bundle-server.js      ← esbuild 服务器打包脚本
├── src/
│   ├── main.js               ← Electron 主进程
│   └── preload.js            ← 预加载脚本
└── assets/
    ├── icon.ico              ← Windows 图标
    ├── icon.png              ← 通用图标
    └── icon.icns             ← macOS 图标
```
